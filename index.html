# Introduction to Node.js ##### Dylan Moore
## First, Some Clarifications.. * Most of Bernhardt's talk is speculation on what **could** happen in order to honestly discuss JavaScript's impact on software and the internet * METAL is fictitious, but asm.js is (and the implications raised in the talk are) real
## And asm.js is...? * http://en.wikipedia.org/wiki/Asm.js * asm.js is an intermediate programming language designed to allow computer software written in languages such as C to be run as web applications while maintaining performance characteristics considerably better than standard JavaScript, the typical language used for such applications.
## Will asm.js Improve My JavaScript Performance? * Stack Overflow user **Bergi**: http://stackoverflow.com/a/15632482 * \> *Not really, because of its dynamic nature. It's the same problem as when trying to compile it to C or even to native code - you actually would need to ship a VM with it to take care of those non-static aspects.*
## Can or Can't We Convert Everything to JavaScript? * There **are** utilities that convert C/C++/Java/Python/etc. code to browser-runnable JavaScript/HTML * **emscripten** - C/C++ to as.js * http://kripken.github.io/emscripten-site * **XMLVM** - .NET/Java/Ruby --> JVM Bytecode --> iOS/C/Javascript/Java * http://www.xmlvm.org * **But** these tools require porting effort for non-trivial applications
## Will All Apps Eventually Become Web Apps? * Maybe? At this point all we know is that there have been some impressive native-to-JavaScript/HTML ports * **QuakeJs** - Quake3 port that runs in the browser * http://www.quakejs.com/ * **SQLite** * http://kripken.github.io/sql.js/GUI/ * **Python/Ruby/Java/Python/More** - In-Browser Compiler/Interpreters * http://repl.it/languages
